# Claim — TS SDK returns 405 (not 404) on non-POST `/invoke`

Directive for the small-claim worker. Grounded in `main` HEAD `9e55830`. This is the
explicitly-deferred SDK-runtime follow-up noted in the #129 cross-SDK conformance work order
(`docs-internal/work-orders/20260629-cross-sdk-conformance.md`), which fixed the harness but
left the TS behavior divergence for a separate SDK PR.

## Target

`workers/sdk-ts/src/index.ts` (+ `workers/sdk-ts/src/index.test.ts`) and the TS skip flag in
`workers/conformance/fixture.json` in `unluckyathecking/crucible`.

## Problem

`workers/sdk-ts/src/index.ts` returns `404` for a non-POST request to `/invoke` (it bundles
method + path into a single 404), while the Go SDK (`crucible.go`) and the Rust SDK both return
`405`. The cross-SDK conformance fixture (`workers/conformance/fixture.json`) documents this as
a known divergence and **skips** the TS method-guard case — so the contract is not enforced for
TS today.

## Change

- In `index.ts`, split the request guard so a non-POST request to the `/invoke` path returns
  `405 Method Not Allowed` with an `Allow: POST` header (matching Go/Rust), while an unknown
  path still returns `404`.
- Update/extend the corresponding assertion in `index.test.ts`.
- Remove the TS skip for the method-guard case in `workers/conformance/fixture.json` so the
  shared fixture now enforces 405 across all three SDKs.

## Expected outcome / acceptance

- A non-POST request to `/invoke` via the TS SDK returns HTTP `405` with `Allow: POST`; an
  unknown path still returns `404`.
- The TS method-guard conformance case is no longer skipped in `fixture.json` and passes.
- `index.test.ts` asserts the 405 + `Allow` header; TS SDK tests green.
- The Go and Rust conformance runs remain green (no fixture regression).

## Constraints

- Stay within `workers/sdk-ts/**` and the single TS skip flag in `workers/conformance/fixture.json`.
- Do not change the Go or Rust SDK runtime, the gateway, or the frozen proto.
- Do not alter any other fixture case or the harness structure (#129) beyond un-skipping this
  one TS case.
- Keep the existing `/healthz` and POST `/invoke` behavior unchanged.
