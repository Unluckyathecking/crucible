# Support

Crucible is a community-maintained framework and does not include guaranteed response
times or commercial support.

Before asking for help:

1. Read the [README](README.md), [adaptation guide](ADAPT.md),
   [API reference](docs/api-reference.md), and
   [compatibility matrix](docs/compatibility.md).
2. Search existing issues and discussions.
3. Reduce the problem to a reproducible example against the latest `main` commit.

Use GitHub Discussions for design and usage questions when Discussions is enabled.
Otherwise, choose the support form from the
[issue chooser](https://github.com/Unluckyathecking/crucible/issues/new/choose).
Include your pinned commit, operating system and architecture, relevant toolchain
versions, configuration with secrets removed, expected and actual behavior, and the
smallest reproduction or logs.

Use the bug form only for reproducible Crucible defects. Use the feature form for a
specific, product-neutral proposal. Security vulnerabilities must follow
[SECURITY.md](SECURITY.md) and must never be posted publicly.
