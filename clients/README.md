# Consumer client SDKs

Type-safe SDKs that call the Crucible Gateway from Go, TypeScript, and Python.
They are generated from `clients/openapi.json`, a committed snapshot of the
gateway's served OpenAPI document, so the SDKs never drift from what the
gateway actually exposes at `GET /openapi.json`.

## Layout

- `openapi.json` — the snapshot. Canonical input for generation.
- `go/` — standalone Go module (`Client`, typed `APIError`, stdlib only).
- `typescript/` — strict-TS npm package (`Client`, `ApiError`, `fetch` only).
- `python/` — pip-installable `crucible_client` package (`Client`, `ApiError`,
  stdlib only; named `crucible_client` so it can't collide with the worker
  SDK's `crucible` package).

## Generated vs hand-maintained

The generator emits, in each language, the request client, the error type, and
the package index — do not edit these by hand:

- Go: `client.go`, `errors.go`, `client_test.go`, `go.mod`, `go.work`
- TypeScript: `src/client.ts`, `src/errors.ts`, `src/index.ts`, `test/client.test.ts`
- Python: `crucible_client/client.py`, `crucible_client/errors.py`,
  `crucible_client/__init__.py`, `tests/test_client.py`

Every generated file except `go.work` carries a `Code generated by
scripts/gen-clients.sh — DO NOT EDIT` header. The `webhook` and `jobs` modules
in each language (and their tests) are hand-maintained and safe to edit.

## Regenerating

```bash
# 1. Refresh the snapshot from the gateway's served document:
go run ./gateway/cmd/spec-dump > clients/openapi.json

# 2. Regenerate all three SDKs from the snapshot (idempotent):
bash scripts/gen-clients.sh

# 3. Confirm nothing drifted:
git diff --exit-code clients/
```

## Adding an endpoint

Register the route in `gateway/internal/server/routes_table.go`, then run the
two regeneration commands above and commit `clients/openapi.json` alongside the
regenerated SDKs.

## CI drift guard

`.github/workflows/client-sdk-drift.yml` re-runs `spec-dump` and fails if the
committed snapshot differs from the served document, regenerates the SDKs and
fails on any `git diff`, then builds and tests all three clients. A spec change
pushed without regenerating the SDKs fails the build.
