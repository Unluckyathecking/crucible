# Governance

Crucible currently uses a maintainer-led governance model.

## Maintainer

- [@Unluckyathecking](https://github.com/Unluckyathecking) — project owner and final
  decision-maker

The maintainer triages issues, reviews and merges changes, manages releases, and makes
final calls on roadmap, compatibility, security, and project policy. Decisions should
be explained in the relevant issue or pull request. Community input is welcome and
technical decisions favor product neutrality, operational safety, backwards
compatibility, and maintainability.

## Becoming a maintainer

There is no fixed contribution count. A contributor may be invited after sustained,
high-quality participation, sound review judgment, respectful community conduct, and
demonstrated care for the whole project. Repository access is granted and revoked by
the project owner.

## Changes to governance

Governance changes are proposed through a pull request to this file and should receive
public community review before adoption. Until additional maintainers are appointed,
the project owner resolves deadlocks.
